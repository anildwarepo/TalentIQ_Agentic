## Design Decisions

**Fail-fast ordering:** Prefix test files `test_0N_` (01, 02, 03, ...) so pytest's default alphabetical sort runs them sequentially. Earlier failures block later tests — if Postgres is down, skip MCP and frontend tests.

**Zero hardcoded values:** All endpoints/names sourced from `azd env get-values` (conftest fixtures). Validated against source of truth:
- `graph_name=talent_graph` from env var
- `embedding_dim=1536` verified in schema inspection
- `FTS_table=employee_fts.fts_vector` column name checked at runtime
- `vector_table=employee_embeddings.resume_embedding` column name checked at runtime
- Proxy path `/af/` confirmed in backend NDJSON response endpoint

**Container exec avoidance:** No `az containerapp exec` (requires TTY + VNet access). Instead:
- Status check via `az containerapp show` (name + resource group)
- Foundry chat completion: deployer credentials (from `azd auth` context)
- Cypher/vector/FTS: via Brett's `db.connect()` (same path pipeline uses)

**Open note:** Backend must expose `/health/foundry` endpoint for production-safe deployment verification (not using deployer creds for health checks).
