# US-007: Administrator maintenance of EQF/MECES mapping table

> **Epic:** [EPIC-03](../epics/EPIC-03.md) — EQF / MECES Study Mapping  
> **Priority:** P1 (Must)  
> **Story Points:** 3  
> **Sprint:** Sprint 2  
> **Status:** New  
> **Business Requirement:** BR-EQF

## Description

As an **administrator**, I want to add, edit, and deactivate entries in the EQF/MECES mapping table, so that the platform can accommodate new qualifications, institutional titles, or country-specific equivalences not covered by the standard pre-load.

## Acceptance Criteria

- [ ] Given I have administrator role, when I access the EQF/MECES mapping management screen, I can view all existing entries with their EQF level, MECES level, scope, and status
- [ ] I can add a new qualification entry providing: qualification name, EQF level (1–8), MECES level where applicable, country/framework scope, and a justification note
- [ ] I can edit any existing entry; changes are saved with a timestamp and the editor's user ID for audit purposes
- [ ] I can deactivate (soft-delete) an entry; deactivated entries are hidden from employee mapping but retained in the audit log
- [ ] Official pre-loaded entries can be edited but are flagged as 'Modified from official source'
- [ ] All changes are logged in an audit trail accessible to administrators
