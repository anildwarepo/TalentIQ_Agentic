# US-039: Soft hold candidate for a specific deal with auto-expiry

> **Epic:** [EPIC-11](../epics/EPIC-11.md) — Soft Hold (Optional)  
> **Priority:** P3 (Could)  
> **Story Points:** 3  
> **Sprint:** Backlog  
> **Status:** New  
> **Business Requirement:** BR-10

## Description

As an **Account/Delivery Manager**, I want to put a 'soft pause' on a candidate for a specific deal or project, with an auto-expiry period so that candidates are not indefinitely reserved when a deal does not progress, so that coordination of personnel is clearer and the bench remains accurate.

## Acceptance Criteria

- [ ] When I place a soft booking (with permission), the candidate appears as reserved with who/when/why metadata
- [ ] When placing a soft hold, the user can set an expiry duration (default configurable by admin; e.g. 30 days)
- [ ] When the expiry date is reached, the soft hold is automatically released and the candidate becomes available again
- [ ] The candidate's manager and the hold owner are notified 48 hours before auto-expiry
- [ ] Auto-expiry events are recorded in the soft-hold history (US-040)
