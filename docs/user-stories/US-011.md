# US-011: Define multiple positions in a single bid search session

> **Epic:** [EPIC-04](../epics/EPIC-04.md) — Candidate Search & CV Search  
> **Priority:** P1 (Must)  
> **Story Points:** 3  
> **Sprint:** Sprint 3  
> **Status:** New  
> **Business Requirement:** BR-01

## Description

As a **Manager/Account Lead**, I want to define multiple positions in a single bid search session (e.g. Position 1 PM, Position 2 Developer), so that I can manage the different requirements of the roles efficiently.

## Acceptance Criteria

- [ ] When Position A and Position B are created with different filters, the app stores and displays filter sets and results for each position separately
- [ ] Results show candidates with potential to complete each position
