# US-010: Apply additional filters on top of search results

> **Epic:** [EPIC-04](../epics/EPIC-04.md) — Candidate Search & CV Search  
> **Priority:** P1 (Must)  
> **Story Points:** 3  
> **Sprint:** Sprint 3  
> **Status:** New  
> **Business Requirement:** BR-01

## Description

As a **Manager/Account Lead**, after running a candidate search I want to apply additional filters on top of the obtained results — for example by city/location (e.g. Avilés, Madrid), but also by any other available attribute — so that I can progressively narrow the shortlist without having to re-run the original search from scratch.

Location is just one example of refinement; the same mechanism must work for any filterable attribute (provider, service line, job level, MECES/EQF level, language proficiency, certification status, etc.).

## Acceptance Criteria

- [ ] Given a result set is displayed from an initial search, when I apply one or more additional filters, the result set is refined in place to show only candidates matching both the original criteria and the additional filters
- [ ] Additional filters can be applied across any available attribute, including but not limited to city/location, provider, service line, job level, MECES/EQF level, language, and certification status
- [ ] Applied additional filters are clearly listed and can be individually removed without re-running the original search
- [ ] Example use case: after searching candidates with PMP certification, I can refine results to only those located in Madrid or Avilés, to comply with bidding restrictions requiring local presence
- [ ] The refined result set count is shown alongside the original result count for transparency
